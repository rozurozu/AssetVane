# Roadmap（開発ロードマップ）

薄い「縦スライス」から始め、横に太らせていく。各 Phase に**完了条件**を置き、満たすまで次へ進まない。

---

## Phase 0: 縦スライス（Vertical Slice）

**目的**: 認証・DB・API・UI を細く 1 本貫通させ、全部品の繋ぎ方を体得する。

- backend に最小の FastAPI を立てる。
- `JQuantsAdapter`（V2 / `x-api-key`）で**数銘柄**の日足を取得。
- SQLite（WAL）に `stocks` / `daily_quotes` を保存。
- FastAPI が「指定銘柄の日足を返す」REST エンドポイントを公開。
- Next.js が API を叩いて**株価チャート**を表示。

**完了条件**: ブラウザで「3 銘柄の株価チャート」が表示され、データが SQLite 経由で FastAPI から来ている。

> いきなり全 4000 銘柄バッチに行くと、認証・DB・UI を同時に相手にして折れる。まず 3 銘柄でフルスタックを貫通させる。

---

## Phase 1: Trend Vane（短期モメンタム検知）— 機能①②

**目的**: 全銘柄バッチとスクリーニングを成立させる。

- 全銘柄の日足を夜間バッチで取得（初回バックフィル + 差分取得、`fetch_meta`）。
- TA-Lib でテクニカル指標（移動平均上抜け、RSI 反転 など＝①）。
- 出来高急増シグナル（直近の出来高が平均比で急増＝②）。
- 結果を `signals` テーブルに保存。
- スクリーニング結果の一覧画面。

**完了条件**: 夜間バッチが全銘柄を処理して `signals` を更新し、一覧画面で「今日の強い銘柄」が見られる。

> Free プランは 12 週間遅延のため、ここで見えるのは約 3 か月前の「強い株」。ロジックは正しく、実運用で Light 以上に切り替えれば最新データで動く（[decisions.md ADR-008](decisions.md)）。

---

## Phase 2: Portfolio Optimizer（資産比率最適化）— 機能④⑤⑥

**目的**: 複数銘柄をまとめて扱い、数理最適化を導入する。実用性が一気に上がる。

- ④ 保有銘柄の相関ヒートマップ。
- ⑤ PyPortfolioOpt による平均分散最適化（リバランス比率提案）。
- ⑥ ポートフォリオ・バックテスト（TOPIX 等との比較）。
- `portfolios` / `holdings` の入力 UI。

**完了条件**: 保有銘柄を入力すると、相関マップ・最適比率・過去シミュレーションが画面に出る。

---

## Phase 3: AI Alpha Scorer（決算スコアリング）— 機能③

**目的**: 機械学習を組み込む。最も難易度が高い。

- `financials` と将来リターンを使って LightGBM の特徴量を設計。
- **学習は別 PC** で実行し、`.pkl` をラズパイにコピー（[decisions.md ADR-006](decisions.md)）。
- ラズパイは推論のみ。スコアを `signals`（`signal_type = ai_alpha`）に保存。
- スコアランキング画面。

**完了条件**: 学習済みモデルでスコアが算出され、ランキング表示される。学習の再現手順がドキュメント化されている。

---

## Phase 4: Signal Beacon（通知）— 機能⑦⑧

**目的**: 画面を開かなくても重要な変化を受け取れるようにする。

- ⑦ 定期リバランス・アラート（前回見直しから一定期間経過で通知）。
- ⑧ 急変動・ブレイクアウト通知（高スコア銘柄・出来高異常を検知時に通知）。
- **Discord Webhook** で送信（LINE Notify は終了済みのため不採用＝[decisions.md ADR-007](decisions.md)）。
- cron スケジュール（例：毎朝の相場開始前）。

**完了条件**: 条件合致時に Discord へ自動でメッセージが届く。

---

## Phase 5: Sector Lead-Lag（日米業種リードラグ）

**目的**: 研究ベースの戦略シグナルを 1 つ追加する。提示のみ（自動売買しない＝[decisions.md ADR-009](decisions.md)）。

**背景**: 「部分空間正則化付き主成分分析を用いた日米業種リードラグ投資戦略」（中川慧ほか, 人工知能学会 SIG-FIN-036, 2026）。米国市場が先に引け、その業種別ショックが翌営業日の日本市場（日中リターン）に遅れて波及する効果を、経済的に意味のある事前部分空間へ正則化した PCA で低ランク予測器として捉える。日足のみ・軽量計算（28×28 程度の相関行列の固有分解）で、ラズパイの夜間バッチに適合する。

- `UsEtfAdapter` で米国 SPDR 業種 ETF の日足を取得（[architecture.md 3](architecture.md)）。
- 日本側は TOPIX-17 業種別 ETF（J-Quants で取得可）。
- 日米結合相関行列を事前部分空間へ正則化 → 固有分解 → 翌日の日本業種スコアを算出。
- スコアを `signals`（`signal_type = lead_lag`）に保存し、業種ランキングを画面提示。

**完了条件**: 「翌日強含む日本業種ランキング」が夜間バッチで算出され、画面に出る。

**留意点**: 論文は取引コスト控除後の超過収益の有無を明示していない。提示用途では軽視できるが、将来の実弾運用ではこの検証が必須。

---

## 進め方の原則

- 各 Phase は前の Phase の完了条件を満たしてから着手する。
- Phase 2 のバックテスト基盤ができてから Phase 5 に進むと、戦略検証がやりやすい。
- 「動く最小」を常に保ち、壊れたら STOP して再計画する。

## 0. 結論（先に）

- **「ai-berkshire で AssetVane を代替」は不可**。両者は性格が異なる。
  - **ai-berkshire** = バフェット/マンガー/段永平/李録の 4 大バリュー投資家の方法論を体系化した **Claude/Codex のスキル集**。CLI を手動起動して 1 銘柄ずつ **Markdown リサーチレポートを生成**する。永続化・自動監視・保有管理は無い。
  - **AssetVane** = 常時稼働 Web アプリ（FastAPI+Next.js+SQLite・夜間全自動巡回・DB 永続・保有資産統合・Discord 能動通知・Tool Calling 提案）。
- **正しい読みは「ai-berkshire の"判断規律の方法論"を AssetVane のパイプラインに移植する」**。ai-berkshire が磨いているのは "深く考える規律"、AssetVane が持つのは "回し続けるインフラ"。移すのは**思想であって形式ではない**。

## 2. ai-berkshire の構成（事実）

| 要素 | 中身 |
|---|---|
| skills/（19）＋ codex-prompts/ ＋ codex-skills/ | 同一方法論を Claude 版・Codex 版で**二重提供**。`scripts/sync-codex-*.py` で同期。codex 側のみ `investment-memo-craft` が 1 本多い |
| tools/（Python・CLI） | `financial_rigor.py`（財務検算）・`momentum_backtest(_v2).py`・`stock_screener.py`・`morningstar_fair_value.py`・`ashare_data.py`・`xueqiu_scraper.py`・`report_audit.py`。argparse の単発実行前提、常時稼働サービスではない。単体 pytest は薄い |
| reports/（約 197 社） | 生成済み投資研究レポート。A株・港股が主力、米株は `美股召回池` に集約、**日本株はほぼ無し** |
| 实盘记录/ | 実売買 log ＋ミラーテスト（美团/PDD） |
| 実績表記 | 2024 +69.29% / 2025 +66.38%（※過去実績・検証不能・本判断には使わない） |

### コアの 3 スキル
- **investment-team**（最重量）: メインが**4 サブエージェントを並列起動**。段永平＝ビジネスモデル質／バフェット＝財務・バリュエーション／マンガー＝逆張り・リスク潰し（invert）／李録＝長期確実性。各々が独立に Web 検索・採点し、**統合時に矛盾をわざと残す**（例：バフェット 4.4=割安 vs 李録 2.0=長期不確実）。
- **quality-screen**: **即死チェックリスト（8 レッドライン）を先に通し 1 つでも該当なら即除外**→ 残りを 5 軸（能力の輪/優れたビジネス/モート/経営陣/安全域）で星取り。合格/条件付き/グレー/不合格の離散判定。
- **portfolio-review**: 各保有の投資テーゼが今も成立するか再検証 → 継続/追加/削減/売却。

### 方法論の核（バイアス対抗メカニズム）
1. **即死チェックリスト（8 レッドライン）を"バリュエーション評価の前段"に置く** — 安さの誘惑を構造的に遮断。
2. **4 視点の独立採点＋矛盾の明示** — 単一 AI の「一方では〜他方では〜」を、4 つの強い意見の衝突に置換。
3. **判定の強制** — 必ず離散ラベル（買い/様子見/回避）＋**三段構え価格帯**（アグレッシブ/標準/保守）を出させる。
4. **マンガー式 invert（逆説テスト）** — 「どうすればこの会社は潰れるか」を確率付きで列挙。
5. **情報リッチネス A/B/C 評価** — 入力データの量と質を先に格付けし「データ少＝確信度を下げる」を強制。
6. **ミラーテスト** — 「5 文で説明できなければ買わない」。言語化強制で理解の浅さを炙り出す。
7. **知的誠実さ** — 「分からない」を許容、推測で確信度を埋めない。
8. **財務検算（AI に暗算させない）** — 指標を**外部の報告値と再計算突合**＋**2 ソース以上のクロスバリデーション**＋**通貨取り違え検出**（HKD/CNY/USD）。`decimal.Decimal`（prec=28）で float 排除。

### 想定パイプライン（スキル間連携）
```
[発掘] industry-research / industry-funnel / bottleneck-hunter
   ↓ 候補
[選別] quality-screen（8 レッドライン＋5 軸）
   ↓ 合格
[深掘] investment-team（4 視点フル） ← investment-research（軽量版）
   ↓ 補完  earnings-review/team・management-deep-dive・news-pulse
[記録] thesis-tracker（前提・反証条件を構造化記録）
   ↓
[監視] thesis-drift（記録と現実を突合・反証条件に触れたらアラート）・portfolio-review
   ↓
[発信] wechat-article
```

## 4. 取り込まない / 合わないもの

- **データ層まるごと**：`ashare_data.py`・`xueqiu_scraper.py`（A株/雪球スクレイピング）は日本株中心の AssetVane に無関係。J-Quants/yfinance/EDINET を既に持つ。
- **スキルの二重メンテ構成・レポート生成 UX**：AssetVane は「レポートを人が読む」でなく「提案を DB に貯め画面/通知で出す」形。移すのは思想であって形式ではない。
- **段永平口調生成・WeChat 記事化**：用途外。
- **謳われた実績（+69%/+66%）**：過去実績・検証不能。取り込み判断の根拠にしない。

## 5. 推し（費用対効果順）

**#1 即死チェックリスト → #3 提案の品質ゲート → #2 skeptic の多面化** の 3 つが、AssetVane の思想（AI に数値を計算させず構造化規律を課す＝ADR-014/016、承認制＝ADR-009）と最も噛み合い、既存資産（skeptic 面・conviction・invalidation・advisor_turns）の自然な延長として小さく載せられる。

- 特に **#1 は quant 純関数＋提案前ゲートで完結**し、ADR-088 の position_review と同じ「決定論で先に効く」パターンなので **dev の LLM 未設定というボトルネックを迂回して価値が出る**。着手候補としては最も手堅い。
- **次アクション**：着手する項目を選定し、`/grill-me` で設計を詰め、ADR＋ATDD に落とす（本リポジトリの標準フロー）。

## 1. 挙動に効くバグ（High・最優先）

### ☑ C-1: 米株/FX 差分取得 — 「0 行＝休場の正常」前提とアダプタの「0 行＝raise」契約が矛盾（**週末ごとに偽の失敗通知**）→ **2026-06-12 実装済み**（`fetch_us_quotes`/`fetch_fx_rates` に `_REFETCH_OVERLAP_DAYS=5` を導入・アダプタ契約「0 行＝raise」を正に docstring 一本化・FX の到達不能分岐は契約違反ガードへ整理）

- **裏取り済み**。`batch/jobs/fetch_us_quotes.py:10` の docstring は「土日も from に含めてよい・yfinance が空で返す」前提だが、`adapters/us_equity.py:211` / `adapters/fx.py:147` は空 DataFrame で `UsEquityAdapterError`/`FxAdapterError` を**送出**する契約（ADR-018「黙って 0 行にしない」）。
- 差分カーソルは「取得できた最大 date＋1」起点のため、**日曜・月曜 02:00 JST の夜間バッチは取得範囲が土日のみ → 全シンボル例外 → 総崩れ `ok=False`**。毎週末 2 晩 × 2 ジョブの偽 Discord アラート＋約 6000 シンボル × 1 秒スロットルの空振りで**夜間約 100 分を浪費**。`fetch_fx_rates.py:92` の「0 行なら休場としてカーソル前進」分岐は**到達不能のデッドコード**。
- Phase 7(B) 実装（6/9・6/11）から最初の週末を跨いでいない場合は未顕在の可能性あり。
- **修正方針**: 同型問題を `fetch_index.py:25` が `_REFETCH_OVERLAP_DAYS=5`（カーソルから数日重ねて取り直し）で解決済み。これを `fetch_us_quotes` / `fetch_fx_rates` にミラーする（UPSERT 冪等なので安全）。あわせてジョブ docstring とアダプタ契約のどちらが正かを一本化する。

### ☑ C-2: 02:00 JST 実行＝米国ザラ場中の「途中足」が確定値として恒久保存される（C-1 と同根）→ **コード側は 2026-06-12 実装済み**（C-1 の overlap で翌晩に確定足 UPSERT 上書き＝自己修復・意図を `_REFETCH_OVERLAP_DAYS` コメントに明記。**実機 `us_daily_quotes` と Yahoo 確定値の突合・汚染分の再バックフィルは運用作業として残**）

- 02:00 JST は米国市場の取引時間中（22:30〜05:00 JST 帯）。yfinance の日足レンジ取得は**進行中セッションの部分バー**を含むため、当日の米国バーが未確定のまま `us_daily_quotes` に保存され、カーソルがその日付まで前進する結果**翌晩以降も補正されない**（`fetch_us_quotes.py:102,124` / FX も同様）。
- 影響: `us_valuation_snapshots`（PER 等）・`asset_snapshots` の米株評価・`lead_lag` の米国側が「深夜 2 時のスナップショット値」を close として使い続ける。
- **修正方針**: C-1 の overlap 再取得で翌晩に確定足で UPSERT 上書き＝自己修復する。導入時に**実機の `us_daily_quotes` と Yahoo 確定値を一度突合**し、汚染分は再バックフィルすること。

### ☑ C-3: `LeadLagWidget` が縮退データ（signal=null）で Dashboard ごとクラッシュ → **2026-06-12 実装済み**（`LeadLagRow.signal` を `number | null` に修正・widget は null を中立色 `text-ink-subtle` の "—" 表示＝`fmtRatio` 流用）

- **裏取り済み**。backend は `LeadLagRankItem.signal: float | None = None`（`routers/lead_lag.py:38`・「縮退時 None」）なのに、`frontend/src/lib/api.ts` の `LeadLagRow.signal` は `number` 非 null、`components/lead-lag/LeadLagWidget.tsx:77-80` は `row.signal.toFixed(2)`・`row.signal >= 0` を直接呼ぶ。null が来ると error boundary が無いので **Dashboard 全体が render エラー**。
- **修正方針**: `signal: number | null` に直し、widget 側で null は "—" 表示。

### ☑ C-4: `fetch_us_fundamentals` — 空 `.info` で既存財務値が NULL 上書きされ `ok=True` でカーソル前進 → **2026-06-12 実装済み**（主要キー全欠損で `UsEquityAdapterError` を raise＝quotes と契約対称。raise 銘柄は既存のシンボル境界 except で partial UPSERT もローテカーソル前進もしない）

- **修正方針**: 主要キー（company_name/eps/shares_net 等）が全欠損なら `UsEquityAdapterError` に倒して quotes と対称化。

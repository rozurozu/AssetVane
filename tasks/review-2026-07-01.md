## 1. 機能不全（defect）— 修正済み

### ☑ #1/#7/#8（High 族）: 共通 `_upsert` が別ジョブの焼いた列を毎晩 NULL で潰す
`db/repo/_common._upsert` は「衝突キー以外の**全列**を EXCLUDED 更新」する仕様。呼び出し側の行 dict に含まれない列（＝別ジョブが分担して書く列）が毎晩 NULL に潰れていた。
- **#1**: `calc_valuation`（毎晩・全銘柄）が `calc_receivables_inventory`（7 日 cadence）の焼いた DSO/DIO・受取債権/在庫 YoY を潰す → ADR-064 #2 が 7 晩に 1 晩しか生きない。
- **#7**: 米株 `calc_us_valuation` も同型で潰す。
- **#8**: `sync_master`（NIGHTLY 先頭）の `upsert_stocks` が `stocks.edinet_code` を毎晩潰す → edinetdb sweep の成果破棄＋API 無駄叩き。
- **修正**: `_upsert` に `partial=True`（rows に実在する列の和集合だけ更新・他列温存）を追加し 3 本へ適用。既に手書き partial だった `upsert_us_stocks` も `_upsert(partial=True)` へ集約（重複解消）。**ATDD**=`tests/test_partial_upsert.py`。**ADR-064 に修正メモ追記**。

### ☑ #9: `POST /cards`・`/cards/{id}/assist` の即時埋め込みが 100% 失敗（asyncio.run を実行中ループ内で呼ぶ）
async ルータで `embed_card_best_effort`（内部 `asyncio.run`）を呼ぶと RuntimeError → 握り潰しで即時埋め込みが常に死に、追加カードは夜間まで意味検索に載らなかった。**修正**=`embed_card_best_effort_async`（`await embed_texts`・services/news.py 同型）を新設し async 経路は await、sync 経路（`PUT /cards`）は薄いラッパで維持。ATDD=`test_cards_api.test_create_runs_immediate_embedding_on_async_path`。

### ☑ #4: `fetch_us_quotes` が書き込み失敗でもカーソル前進＋総崩れ判定漏れ（サイレント欠落）
`max_date` を UPSERT 前に積み、書き込み失敗後もその date までカーソル前進＝未永続データを飛び越え。書き込み失敗は 1 バッチ 1 エントリで `len(failed) < attempted` となり全滅でも ok=True・無通知。**修正**=カーソルは**永続化に成功したバッチの最大 date だけ**から積む・書き込み失敗は symbol 単位で failed に積む。ATDD=`test_us_quotes_job` に 2 件。

### ☑ #5: tagger 面未設定で `tag_news_polarity` が毎晩 Discord 誤アラート（沈黙 skip 契約違反）
`classify_polarities` が面未設定（沈黙 skip）と LLM 総崩れをどちらも空 dict に潰し、ジョブが総崩れ扱い → ok=False → 誤通知。**修正**=`classify_polarities` は `FaceNotConfiguredError` を握らず伝播、ジョブが「面未設定＝沈黙 skip（ok=True）」と「総崩れ＝通知」を切り分ける（embed_news の embedding_enabled() 事前ガードと同型）。ATDD=`test_news_polarity.test_tag_news_polarity_unconfigured_face_is_silent_skip`。

### ☑ #2: `screen_us_valuation` が gics_sector を正規化せずサイレント 0 件
正式 GICS 名（'Information Technology'）を渡すと格納値 'Technology' と exact 一致せず 0 件＝「該当なし」誤結論。**修正**=`handle_screen_by_theme` と同様に `normalize_gics_sector` を適用。ATDD=`test_us_valuation_tools.test_screen_us_valuation_normalizes_formal_gics_name`。

### ☑ #3: active な知識カードを「AIで整える」で再整形すると draft 降格（AI 注入から静かに脱落）
`reassist` が現在 status を見ず、AI が verdict='active' を返すと status=draft（active化は人間承認）に上書き＝active カードを無言降格。**修正**=現在 active かつ verdict=='active' なら active 温存。ATDD=`test_cards_api.test_reassist_active_card_is_not_demoted`。

### ☑ #6: AI Alpha `surprise_proxy` の train/serve skew（学習=開示前窓・推論=開示後込み窓）
`_surprise` の `post = min(idx+window, as_of境界)` が as_of 依存で、学習(as_of=開示日)は開示前窓・推論(as_of=当日)は開示後込み窓＝同一 (code,開示) が別分布。週末開示の非対称もあり。**修正**=アンカーを「開示日以前で最後の営業日」に固定し窓を常に `[idx-window, idx]`（開示前のみ）へ統一＝as_of 非依存・週末 skew も解消。`.pkl` 未配置なので学習実測前に修正。ATDD=`test_quant_ml_features.test_surprise_proxy_consistent_train_vs_serve`。

### ☑ #13: max_rounds 打ち切りのプレースホルダが偽 observations 化（縮退検知素通り＋ゴミ journal）
`run_tool_loop` の打ち切りで content 空時に合成プレースホルダを返すと nightly で reply→observations に化け、縮退検知（ADR-018）を素通りして偽成功＋ゴミ journal。**修正**=実 content があれば返す・空なら nightly は ""（縮退へ倒す）/chat は定型文（UX 維持）。ATDD=`test_advisor_service` に 2 件。

### ☑ #16: 閏日(2/29)実行で backfill 開始日計算が ValueError クラッシュ（5 ジョブ）
`last.replace(year=last.year - N)` が 2/29 → 非閏年で ValueError。**修正**=`_cursor.backfill_start_date`（閏日安全・2/29→2/28 丸め）へ 7 箇所集約。ATDD=`test_cursor.test_backfill_start_date_leap_day_does_not_crash`。

### ☑ #26: `PUT /transactions`・`/fund-transactions` が body.portfolio_id で recalc（実所属と乖離）
update は portfolio_id を更新しないのに body の別 portfolio で recalc すると実所属側 holdings が取引と乖離。**修正**=delete と同型に実所属 portfolio_id で recalc。ATDD=`test_portfolio_api.test_put_transaction_recalcs_actual_portfolio_not_body`。

### ☑ #11: `submit_notable_stocks` が picks 全件一括検証で 1 件不備→全落ち（PLAUSIBLE）
`SubmitNotableStocksArgs.model_validate` を一括実行し 1 件の不備で全 picks 破棄＝ADR-067 の「注目＝AI 提案なし」を再誘発しうる。**修正**=picks を 1 件ずつ `NotablePickArg.model_validate` し不備の pick だけスキップ（coerce_policy_change と同じ per-item グレースフル）。ATDD=`test_notable_persist.test_persist_skips_malformed_pick_keeps_valid`。

### ☑ #28: セクション上部フォームの記録が取引履歴一覧に反映されない（再取得漏れ）
`UsSection`（＋同型の `FundSection`）で上部フォームの onDone は保有を更新するが、履歴コンポーネントの自前 fetch を再取得しない。**修正**=`reloadKey` プロップを親が bump→履歴の useApi deps に含める（履歴の自前 mutation は自己 refetch で bump 不要）。US・投信の両方を修正。

### ☑ #29: 外部資産の削除失敗が握り潰される（try/catch なし）
`AssetInputPanel.handleExternalDelete` に try/catch が無く失敗が無通知。**修正**=try/catch＋`setExternalErr`（他 mutation と対称）。

### ☑ #30: 米株詳細でバリュエーション取得エラーが「データ無し（—）」に化ける
バリュエーション枠が loading/error を無視し常に `v?.field`（—）を出す＝取得失敗が「未焼成」に見える。**修正**=枠を StatusBlock で包み loading/error を表示・ヘッダも状態表示。

### ☑ #19 + ADR-070 doc ドリフト（4 箇所）
`b967bf3e`（ADR-070）で `POST /batch/stop` は running ゲート撤廃で常に stopping=true になったが、router docstring・フィールドコメント・frontend の型コメント・`GET /batch/status` の「CLI --nightly も映る」記述が旧挙動のまま。**修正**=`routers/batch.py`（2 docstring＋1 コメント）・`lib/api/batch.ts`（2 コメント）を新挙動へ同期。

### 見送り（理由付き）
- **#25**（アダプタの数値/日付正規化ヘルパ重複）: `_to_float`/`_norm_yahoo_date`/`_first` は jquants/fx/index/us_equity に**意図的差分**がある（jquants の `_to_float` は fins/summary 対策で `''`→None、fx の `_norm_yahoo_date` は `str()` fallback ＝ index/us_equity は `_norm_date` fallback）。真に同一ではなく、共通化は正規化のドリフトを招く上、backend-adapter-pattern スキルの「正規化はアダプタ内に閉じ込める」不変条件と競合する。**低価値・高ドリフトリスクにつき見送り**（`_throttle`/`get_with_retry` を共通化した 2026-06-13 の判断で jquants/news を意図的に対象外にしたのと同じ線引き）。

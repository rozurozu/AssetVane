# 外部データソースアダプタ規約

外部 API（J-Quants・指数・米株・ニュース・為替 等）は**アダプタ越しに使う**。router/service/batch から外部 API を直結ハードコードしない（ADR-010）。アダプタは「外の世界の都合」を 1 ファイルに閉じ込め、内側には安定したデータだけを渡す。

## アダプタの責務

- **外部 API へのアクセスを一手に引き受ける**。HTTP・認証・ページング・リトライ・スロットルをアダプタ内に隠す。呼び出し側は `code` や期間を渡すと正規化済みの行が返る、という契約だけ見る。
- **「外部キー名 → 内部列名」の対応をこのファイルに閉じ込める**。DB の列は安定した内部名のまま保ち、外部 API のフィールド名（略記・命名変更・バージョン差）を内側に漏らさない。
- データソースごとに 1 アダプタ（`JQuantsAdapter` / `IndexAdapter` / ...）。
- **参照知識（業種コード等の対応表・ラベル・正規化定数）は外部 IO ではないので adapters に置かない**＝`app/reference/`（[[backend-foundations]] の序列外・中立横断）へ置く。adapters は**外部 API アクセスと「外部キー名→内部列名」の正規化**に限る。

## 複数ソースのフォールバック（同じ事実を冗長化する場合）

1 つの事実（例: 指数のある日の終値）を**複数ソースから冗長に取りたい**ときは、ソース別の小クラス＋ファサードに分ける（`IndexAdapter` が実例・ADR-010・grill 2026-06）。

- **ソース別クラス**（`StooqIndexSource` 等）が共通基底（`IndexSource`）を実装し、各々が自分の取得・パース・スロットル・**応答検証(ガード)**を持つ。ソース固有の検証（例: Stooq の「CSV ヘッダで始まらなければ raise」）は**そのソースのクラスに閉じ込める**（汎用ファサードに置かない）。
- **ファサード**（`IndexAdapter`・名前は呼び出し側互換のため維持）が、設定（`settings.index_sources` の CSV・優先順）を**名前→クラスのレジストリ**で解決し、優先順に試す。**ソースが例外なら次へ・成功（0 行＝正規の空 含む）なら採用して打ち切り・全ソース失敗なら独自例外を raise**（→ batch が ok=False → 通知）。
- **ソースをまたぐ正規化**: 内部・DB は canonical な識別子で統一し（例 `index_quotes.symbol`）、ソース別の表記差は各ソースクラスが内部で吸収して canonical に戻す（フォールバックを透過に・スキーマに source 列を増やさない）。
- **状態はインスタンスに持つ**（スロットル時刻等）。モジュールグローバルの可変状態・`asyncio.Lock()` をソース横断で共有しない（イベントループ／プロセス跨ぎで壊れる＝[[batch-pattern]] の `asyncio.run` 連発で踏む）。
- これは「同じ事実を冗長化」する構成。News のように**別々のアイテムを寄せ集める**用途は gather＋URL 重複排除（union）で、こことは別パターン（ADR-020）。

> 単一ソースのうちは普通に 1 アダプタでよい。冗長化が要るソース（ブロック・障害が現実的に起きる）でだけこの形にする（過剰実装を避ける）。

## キー名の正規化（外→内の境界）

外部レスポンスのフィールド名は変わりうる（V1/V2 差・略記）。**候補キーのフォールバック**で吸収し、内部列名に正規化する。

```python
def _first(d: dict[str, Any], keys: list[str], default: Any = None) -> Any:
    """候補キーのうち最初に存在した値を返す（略記/フルネーム両対応）。"""
    for k in keys:
        if k in d and d[k] is not None:
            return d[k]
    return default

# 正規化: 外部キー（候補複数）→ 内部列名（安定）
row = {
    "code": _to_internal_code(_first(raw, ["Code", "code"])),
    "date": _norm_date(_first(raw, ["Date", "date"])),
    "adj_close": _first(raw, ["AdjustmentClose", "AdjC", "adj_close"]),
}
```

- 日付・コード等の形式正規化（`YYYYMMDD`→`YYYY-MM-DD`、4 桁→5 桁等）もアダプタ内のヘルパで行う。
- 正規化後の行は[[backend-repo-pattern]] の UPSERT にそのまま渡せる内部列名にする。

## リトライ・スロットル・ページング

- **スロットル**（レート制限対策）をアダプタ内に持つ。前回リクエストからの最低間隔を `time.monotonic()` で計り `time.sleep` で待つ。間隔は設定（`settings`）から読み、モジュール定数はフォールバック既定にする。
- **リトライ**は 429/一時失敗に指数バックオフ。最大回数を定数で持つ。
- **同期・同型のスロットル／リトライは共通コンポーネントに寄せてよい**。前回間隔（`time.monotonic`＋`time.sleep`）と 429／一時失敗の指数バックオフ（最大回数は定数）は同期アダプタ間で同型になりやすい。これらは `app/adapters/_http.py` のコンポーネント（スロットルは状態を持つ小オブジェクト・リトライは関数）に寄せ、各アダプタ／ソースは**スロットルをインスタンスに保持**して使う（モジュールグローバル可変状態・`asyncio.Lock` をソース横断で共有しない原則は不変）。用途別の独自例外メッセージ（対処を含む）と返り型（`.content`/`.text`/`.json()`）の取り出しはアダプタ側に残す（共通リトライにエラー翻訳のコールバックを渡し、返り値は Response に統一）。
- **特殊事情は個別実装に残す**。クラス変数でジョブ境界のバーストを抑える・通常より多い再試行回数や長い待機上限（プラン制限由来の長時間ブロック耐性）を持つ同期アダプタや、イベントループ事情を抱える async アダプタは無理に寄せず個別実装のままにする（共通化は「同型のものだけ」に限る・過剰実装を避ける）。
- **ページング**（`pagination_key` 等）はアダプタ内で辿って全行を集約し、呼び出し側に見せない。
- エンベロープからの行抽出（`{"data": [...], "pagination_key": ...}`）もアダプタ内のヘルパに。

## 設定・認証・例外

- 認証情報は `settings`（backend の `.env`）から読む。**キーをハードコードしない・frontend に渡さない**。J-Quants は V2（`x-api-key` ヘッダ）を使う（ADR-008）。
- 未設定・取得失敗は**用途別の独自例外**（例 `JQuantsError(RuntimeError)`）で投げ、メッセージに対処（`.env` の設定等）を書く。呼び出し側（router/batch）が翻訳する（router は HTTPException、batch は JobResult/通知）。
- HTTP クライアントは `httpx`。`with httpx.Client(...) as client:` でタイムアウトを設定。

## チェックリスト

- [ ] 外部 API アクセスをアダプタに閉じ込めた（router/service/batch から直結していない）
- [ ] 外部キー名→内部列名の正規化をアダプタ内に閉じ込めた（候補キーのフォールバック・形式正規化）
- [ ] リトライ・スロットル・ページングをアダプタ内に隠した（設定値は `settings` から）
- [ ] 同期の同型スロットル／リトライは `_http.py` のコンポーネントへ（Throttle はインスタンス保持・例外翻訳はコールバック）。特殊事情（クラス変数・多めの再試行・async）は個別実装に残した
- [ ] 冗長化が要るなら、ソース別クラス＋ファサード（優先順フォールバック）で。ソース固有の検証は各クラスに閉じ込め、状態はインスタンスに持つ（モジュールグローバル共有を避ける）
- [ ] 認証情報は `settings` から（ハードコード・frontend 露出なし）。J-Quants は V2 `x-api-key`
- [ ] 失敗は用途別の独自例外（対処を含むメッセージ）。呼び出し側で翻訳
- [ ] 正規化後の行が repo の UPSERT にそのまま渡る内部列名
- [ ] docstring 冒頭に ADR-008/010 参照
